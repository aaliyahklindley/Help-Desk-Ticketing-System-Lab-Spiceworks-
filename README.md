# Key Components of a Ticketing System

Every support ticket typically contains important information used to track and manage IT issues.

| Field | Purpose |
|---|---|
| Ticket ID | Unique identifier for the issue |
| Requester | Person reporting the issue |
| Category | Type of issue (hardware, software, network, etc.) |
| Priority | Determines urgency and impact |
| Assigned To | Technician responsible for resolving the issue |
| Status | Tracks ticket progress |

# Ticket Priority Levels

Priority levels help IT teams determine how urgently issues need to be addressed.

| Priority | Impact | Example |
|---|---|---|
| Critical (P1) | Entire company affected | Company email server outage |
| High (P2) | Department-wide issue | Payroll system unavailable |
| Medium (P3) | Single user affected | Employee laptop running slowly |
| Low (P4) | Minor inconvenience | Software installation request |

---

# Ticket Status Definitions

| Status | Meaning |
|---|---|
| New | Ticket created but not yet reviewed |
| In Progress | Technician actively troubleshooting |
| Pending User Response | Waiting for additional user information |
| Escalated | Transferred to higher-level support |
| Resolved | Issue fixed and awaiting confirmation |

---

# IT Support Ticket Lifecycle

## Standard Ticket Process

1. User submits ticket
2. Ticket is categorized and assigned
3. Technician troubleshoots the issue
4. Issue is resolved or escalated
5. Ticket is closed
6. Optional post-resolution follow-up

---

# Service Level Agreements (SLA)

## What is an SLA?

A Service Level Agreement (SLA) is a formal agreement that defines expected response and resolution times for technical support requests.

SLAs help IT teams maintain service quality and prioritize issues based on business impact.

---

# Key Components of an SLA

- Response time
- Resolution time
- Ticket prioritization
- Escalation procedures
- Service availability
- Breach consequences

---

# Why SLAs Matter

SLAs help:
- Ensure timely support
- Manage user expectations
- Prevent ticket backlogs
- Measure IT team performance
- Standardize support operations
